# 🧑‍💼 Office Operations Guide

## Daily flow

1. Open **Dashboard** to see today, pending, follow-up, and collection health.
2. Use **Students** to add/search/update students and manage exceptions.
3. Use **Fee Setup** to update year defaults and policies.
4. Use **Payment Desk** to collect fees and print receipts.
5. Use **Transactions** to verify receipts, dues, and payment history.
6. Use **Defaulters** for daily follow-up.
7. Use **Exports** for Excel files.
8. Use **Admin Tools** only for rare setup, corrections, and troubleshooting.

## Add or update a student

- Enter student details.
- SR/admission can be blank for add flow; the app generates a pending SR marker.
- Confirm class, section, route, and session.
- Saving should trigger dues preparation automatically.

## Bulk add or update students

Use the staged import workflow:

1. Choose Bulk Add or Bulk Update.
2. Download the correct template.
3. Upload CSV/XLSX.
4. Map columns.
5. Run validation.
6. Review row warnings/errors.
7. Approve valid rows only.
8. Commit and keep batch history.

Identity rules:

- Bulk Add creates new students only.
- Bulk Update updates existing students only.
- Matching priority is Student ID, then SR/admission number.
- Name alone is never a safe update identity.

## Fee Setup flow

1. Open Fee Setup.
2. Confirm session.
3. Update tuition, transport, due dates, late fee, or academic fee values.
4. Preview changes.
5. Review protected rows.
6. Publish only after review.

## Payment collection flow

1. Open Payment Desk.
2. Select class.
3. Select student.
4. Review dues and payable amount.
5. Enter amount or choose quick amount.
6. Choose payment mode.
7. Add reference if available.
8. Confirm and post.
9. Open/print receipt.
10. Use Collect Another Payment for the next student.

## Defaulters follow-up

Use Defaulters for daily calling/follow-up. Filter by:

- class
- route
- overdue pressure
- search

## Exports

Use Exports for:

- student lists
- class-wise dues
- defaulters
- receipt register
- conventional discount reports
- refund/credit views where configured

Exports are office copies. The system record remains the source of truth.
